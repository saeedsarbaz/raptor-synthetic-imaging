## ğŸ’¡ Pro Tips

### Commit Best Practices
```bash
# Good commit messages
git commit -m "Add external validation power analysis to Task 4"
git commit -m "Fix: Correct McNamara citation in references"
git commit -m "Docs: Update README with preliminary CBCT results"

# Bad commit messages (avoid)
git commit -m "updates"
git commit -m "fix stuff"
git commit -m "asdfasdf"
```

### Keep Repository Clean
- Never commit build files (*.aux, *.log)
- Never commit patient data
- Never commit large binaries (>100MB)
- Use .gitignore effectively

### Version Numbering
- **v1.0.0** = Submission-ready
- **v1.1.0** = Add preliminary results
- **v2.0.0** = Major methodology revision
- **v1.0.1** = Typo fixes
